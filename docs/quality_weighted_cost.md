# Quality-Weighted Cost Comparison

Metric definitions:
- **A** = total assertions in the file
- **U** = unique (post-dedup)
- **CU** = complex AND unique (uses `$past` / `$stable` / `$rose` / `$fell` / `$onehot` / `inside` / `|=>` / multi-signal antecedent; not a bare `$bits()` width check)
- **D** = distinct functional domains in the file
- **Q** = CU × D — composite quality unit (penalises trivial inflation, duplicate copies, single-domain output)
- **tok/Q** = total_tokens / Q — *tokens per usable, complex, diverse assertion-unit*. Lower is better.
